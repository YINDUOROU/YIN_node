尹さんのJAVA总结（面向对象基础）

✅面向对象

一、什么是面向对象？

面向对象是一种程序设计思想，把现实世界的事物抽象为“对象”，每个对象有属性（字段）和行为（方法）。

Java 中“面向对象”体现在三个核心特性：

特性中文说明
Encapsulation封装把数据和方法打包成对象，保护内部细节
Inheritance继承子类继承父类代码，代码复用
Polymorphism多态同一方法根据对象类型表现不同行为

二、类与对象（Class & Object）

🔹 定义类

public class Person {
    // 属性（字段）
    String name;
    int age;

    // 方法（行为）
    void sayHello() {
        System.out.println("你好，我是 " + name);
    }
}

🔹 创建对象

Person p = new Person();
p.name = "小明";
p.age = 18;
p.sayHello();  // 输出：你好，我是 小明

三、构造方法（Constructor）

构造方法用于创建对象时初始化属性。方法名与类名相同，无返回类型。

public class Person {
    String name;

    // 构造方法
    public Person(String n) {
        name = n;
    }
}
Person p = new Person("小红");  // 调用构造方法

✅ 成员变量、成员方法（普通方法）、构造函数的区别总览

项目成员变量（字段）成员方法（普通方法）构造函数（Constructor）
作用保存对象的属性值表现对象的行为逻辑创建对象并初始化成员变量
定义位置类中，方法外类中，方法（函数）定义类中，方法名与类名相同
是否有返回值有数据类型有（可以是 void）❌ 没有返回类型（连 void 都不能写）
是否需要调用自动创建、可直接访问必须通过对象调用使用 new 时自动调用
是否可以重载❌ 不可重载✅ 可重载（不同参数）✅ 可重载（不同参数）
是否可继承✅（非 private 可继承）✅（非 private 可继承）❌ 构造函数不继承
是否可以加 static✅（类变量）✅（类方法）❌ 不能加 static
是否可加 final✅（不可再赋值）✅（不可被重写）❌ 构造函数不能加 final

✅ 分别解释与示例

🔹 1. 成员变量（Field / 属性）

定义：

public class Student {
    String name;       // 实例变量
    static int count;  // 静态变量
}

特点：

  • 存储对象的状态（如姓名、年龄）
  • 每个对象有自己的一份（static 除外）
  • 默认值：数字是 0、引用是 null、布尔是 false

🔹 2. 成员方法（普通方法）

定义：

public class Student {
    void study() {
        System.out.println("在学习");
    }

    int getAge() {
        return 18;
    }
}

特点：

  • 表现对象的行为
  • 可访问/操作成员变量
  • 可被调用（如 s.study()）

🔹 3. 构造函数（Constructor）

定义：

public class Student {
    String name;

    // 构造函数
    public Student(String n) {
        name = n;
    }
}

特点：

  • 与类名相同
  • 无返回类型（连 void 也不能写）
  • 用于对象创建时初始化成员变量
  • 可重载（多个构造器）
  • new 时自动调用

✅ 示例对比

public class Person {
    // ✅ 成员变量
    String name;
    int age;

    // ✅ 构造函数
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // ✅ 成员方法
    void introduce() {
        System.out.println("我是 " + name + "，年龄 " + age);
    }
}
// 使用方式：
Person p = new Person("小明", 20);  // 构造函数被调用
p.introduce();                      // 成员方法被调用

✅ 面试常问总结

❓ Q1：构造函数和普通方法有什么不同？

比较项构造函数普通方法
是否有返回值❌ 没有（不能写 void）✅ 有
是否自动调用✅ 创建对象时自动执行❌ 需手动调用
名字是否与类相同✅ 必须相同❌ 自定义

❓ Q2：成员变量和构造函数有什么不同？

  • 成员变量是“对象属性”，构造函数是“创建对象时设置属性”。
  • 变量存状态，构造函数赋初值。

四、封装（Encapsulation）

🔹 用 private 修饰属性，隐藏内部数据

🔹 用 public 提供 getter/setter 方法暴露控制访问

public class Student {
    private int score;

    public int getScore() {
        return score;
    }

    public void setScore(int s) {
        if (s >= 0 && s <= 100) {
            score = s;
        }
    }
}

用于控制“类、方法、变量”等的访问权限范围：

限定符中文解释本类同包子类其他包常用位置
public公共的✅✅✅✅类、方法、变量
protected受保护的✅✅✅❌方法、变量
默认包访问权限✅✅❌❌类、方法、变量（不写时）
private私有的✅❌❌❌方法、变量（不能修饰类）

其他常用修饰符（Modifiers）

🔹 1. static（静态的）

  • 作用：表示属于类本身而不是某个对象
  • 可修饰：变量、方法、代码块、内部类
  • 特点：无需创建对象即可访问
public class MyClass {
    static int count = 0;
    static void say() {
        System.out.println("静态方法");
    }
}

调用方式：

MyClass.say();     // 直接通过类名调用
System.out.println(MyClass.count);

🔹 2. final（最终的 / 常量）

  • 作用：不可变、不可重写、不可继承
  • 可修饰：变量、方法、类
修饰对象含义
变量变成常量，只能赋值一次
方法子类不能重写此方法
类不能被继承

final int MAX = 100;          // 常量
final class A {}              // 不能有子类
final void run() {}           // 子类不能 override

🔹 3. abstract（抽象）

  • 作用：用于声明抽象类和抽象方法
  • 特点：不能被实例化，必须由子类实现
abstract class Animal {
    abstract void speak();    // 无方法体
}

class Dog extends Animal {
    void speak() {
        System.out.println("汪汪！");
    }
}

🔹 4. synchronized（同步的）

  • 作用：用于线程安全，防止并发冲突
  • 可修饰方法或代码块
public synchronized void save() {
    // 线程同步方法
}

🔹 5. transient（瞬时的）

  • 作用：在序列化对象时，该字段不会被序列化
transient String password;

🔹 6. volatile（易变的）

  • 作用：保证线程间变量的可见性
  • 用于多线程开发中共享变量的同步场景
volatile boolean running = true;

五、继承（Inheritance）

🔹 使用 extends 继承父类

class Animal {
    void eat() {
        System.out.println("吃东西");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}
Dog d = new Dog();
d.eat();   // 父类方法
d.bark();  // 子类方法
//子类继承父类，那么子类同样也能继承父类的方法

六、多态（Polymorphism）

多态的表现形式：方法的重写和重载

🔹 向上转型：父类引用指向子类对象

Animal a = new Dog();  // Animal 是父类，Dog 是子类
a.eat();               // 调用的是 Dog 继承的 eat 方法

🔹 重写（Override）：子类重写父类方法

class Dog extends Animal {
    @Override
    void eat() {
        System.out.println("狗在吃饭");
    }
}
//子类继承父类后，重新定义父类已有的方法，方法名、参数、返回值必须完全一致。

🔹 重载（Overloading）：在同一个类中，方法名相同，但参数不同（个数、类型或顺序不同），形成重载。

✅ 特点：

项目内容
是否在同类内✅ 是
参数列表是否不同✅ 必须不同
返回值是否必须相同❌ 不要求相同（不能仅靠返回值区分）
是否与继承有关❌ 无关
何时决定调用哪个方法🕐 编译时决定（静态多态）

✅ 示例：

class Printer {
    void print(String s) {}
    void print(int n) {}
    void print(String s, int n) {}
}

三者对比总结表格（超重要）

项目方法重载（Overload）方法重写（Override）向上转型（Upcasting）
是否要求继承否是是
方法名是否相同是是无要求
参数是否相同❌ 必须不同✅ 必须相同无关
返回值是否可不同✅ 可不同❌ 必须相同或兼容无关
发生位置同一个类父子类之间父类引用 = 子类对象
调用时间决定者编译时（静态多态）运行时（动态多态）编译时看父类，运行时走子类
是否构成多态✅ 是（静态多态）✅ 是（动态多态）✅ 是多态的实现手段

七、this 与 super

关键字说明
this当前对象的引用（访问属性/方法/构造器）
super调用父类的属性/方法/构造器

示例：

class Dog extends Animal {
    void show() {
        System.out.println(super.toString()); // 调父类方法
        System.out.println(this.getClass());  // 当前类名
    }
}

✅ 接口（interface）与抽象类（abstract class）详细讲解

一、抽象类（abstract class）

🔹 1. 什么是抽象类？

抽象类是不能被实例化的类，它用来作为其他类的**“父类模板”**，定义一组共通的属性和行为。

🧠 背后设计思想：

  • 有些类是抽象概念，本身无法独立存在（如“动物 Animal”），但其子类（如“狗 Dog”、“猫 Cat”）可以。
  • 抽象类用于代码复用 + 强制子类实现特定方法。

🔹 2. 语法规则

public abstract class Animal {
    String name;

    public void sleep() {
        System.out.println(name + " 睡觉");
    }

    public abstract void speak(); // 抽象方法，没有方法体
}

🔸 特点详解：

特性说明
是否可实例化❌ 不能用 new Animal()
是否可包含字段✅ 可以定义字段（如 String name）
是否可包含构造器✅ 可以（用于初始化字段）
是否可包含方法实现✅ 普通方法、抽象方法都可以
是否可以继承其他类✅ 可以（包括继承抽象类）
是否支持多继承❌ 不支持多个类的继承

🔹 3. 子类继承抽象类的规则

  • 子类必须实现所有抽象方法
  • 或者子类本身也必须是抽象类
public class Dog extends Animal {
    public void speak() {
        System.out.println("狗：汪汪！");
    }
}

🔹 4. 抽象类的实际应用场景

  • 通用业务逻辑复用（如通用增删改查）
  • 父类定义骨架，子类定义具体逻辑（Template 模式）
  • 框架底层：Spring 的 AbstractController、AbstractApplicationContext

二、接口（interface）

🔹 1. 什么是接口？

接口是一组方法的规范，用于约束类的行为。它不关注“做什么”，而关注“能做什么”。

🧠 设计背景：

  • Java 不支持多继承 → 接口弥补这一点
  • 接口定义“能力”或“行为契约”，例如“USB 接口”“可比较对象 Comparable”

🔹 2. 语法规则

public interface USB {
    // 默认是 public abstract（即使不写）
    void connect();
}

🔸 接口中成员默认特性：

成员默认修饰符
方法public abstract
变量public static final（常量）

🔹 3. 实现接口的类必须如何做？

public class Mouse implements USB {
    public void connect() {
        System.out.println("鼠标连接成功");
    }
}
  • 类可以实现多个接口（implements A, B, C）
  • 一个类最多只能继承一个类（单继承）

🔹 4. Java 8+ 的接口新特性（重要）

特性说明示例
default 方法接口可以提供默认实现default void log() {}
static 方法可定义工具方法static void help() {}
private 方法（Java 9）内部辅助方法private void helper()

示例：

interface Logger {
    default void log(String msg) {
        System.out.println("日志：" + msg);
    }

    static void help() {
        System.out.println("帮助信息");
    }
}

🔹 5. 接口的实际使用场景

  • 面向接口编程（Spring 全是接口驱动）
  • 框架回调机制（如 Runnable 接口、Controller 接口）
  • 策略模式、适配器模式

✅ Java 的异常处理机制（Exception Handling）

这是实际开发和面试中必考知识点，能否写出健壮、安全的代码，取决于你是否真正理解了 Java 的异常系统。

一、什么是异常（Exception）？

异常是在程序运行过程中发生的不正常情况，如除以 0、数组越界、空指针等。

Java 通过 异常类（Exception Class） 来封装错误，并使用 try-catch 机制 捕获处理，避免程序崩溃。

二、异常体系结构图（继承关系）

Throwable（所有异常的顶级父类）
├── Error（严重错误，如内存溢出，程序不能处理 ❌）
└── Exception（一般异常，程序可以捕获处理 ✅）
    ├── 编译时异常（Checked Exception）→ 必须处理
    └── 运行时异常（Unchecked Exception）→ 可选处理

三、异常的分类

类型名称说明示例
Checked Exception（受检异常）IOException、SQLException编译器强制处理，必须 try-catch 或 throws文件找不到
Unchecked Exception（非受检异常）NullPointerException、ArrayIndexOutOfBoundsException编译不强制处理，可选空指针、越界
Error（错误）OutOfMemoryError、StackOverflowError程序无法恢复，不建议捕获内存溢出

四、try-catch-finally 语法结构

try {
    // 可能出异常的代码
} catch (异常类型1 e1) {
    // 异常处理逻辑1
} catch (异常类型2 e2) {
    // 异常处理逻辑2
} finally {
    // 无论是否异常，都会执行
}

示例：

public class Demo {
    public static void main(String[] args) {
        try {
            int a = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("发生算术异常：" + e.getMessage());
        } finally {
            System.out.println("无论异常与否都会执行");
        }
    }
}

输出：

发生算术异常：/ by zero
无论异常与否都会执行

五、throws 与 throw 的区别（非常重要）

关键词说明用法
throw手动抛出一个异常对象throw new NullPointerException();
throws声明该方法可能抛出的异常public void readFile() throws IOException {}

示例：

public void readFile(String path) throws IOException {
    FileReader fr = new FileReader(path);  // 受检异常
}

/*
FileReader 是 Java 提供的内置类，在 java.io 包中。
它用于 按字符方式读取文本文件。
它的构造方法会抛出 FileNotFoundException，所以必须 try-catch 或 throws。
你不需要也不应该自己去定义一个同名类。
*/

调用方必须处理：

try {
    readFile("a.txt");
} catch (IOException e) {
    e.printStackTrace();
}

//完整示例
import java.io.*;

public class FileReaderDemo {

    // 方法定义处：抛出异常
    public void readFile(String path) throws IOException {
        FileReader fr = new FileReader(path);
        int ch;
        while ((ch = fr.read()) != -1) {
            System.out.print((char) ch);
        }
        fr.close(); // 关闭资源
    }

    public static void main(String[] args) {
        FileReaderDemo demo = new FileReaderDemo();

        // 调用处：try-catch 捕获异常
        try {
            demo.readFile("a.txt");
        } catch (IOException e) {
            System.out.println("读取文件时发生错误！");
            e.printStackTrace();
        }
    }
}

图解流程：
你写了一个 readFile 方法 ----> 里面用到了 FileReader ----> 可能抛 IOException（受检）
==> 你必须告诉别人：“我可能会抛异常” ----> 所以方法签名加上 throws IOException
==> 别人在调用你这个方法时，Java 编译器会强制他们加上 try-catch 或继续 throws

六、自定义异常类（面试加分项）

步骤：

  1. 继承 Exception 或 RuntimeException
  1. 提供构造函数
public class MyException extends Exception {
    public MyException(String message) {
        super(message);
    }
}

使用：

if (age < 0) {
    throw new MyException("年龄不能为负数");
}

✅ 七、多个 catch 的执行顺序与异常捕获机制

  • catch 从上往下执行，一旦匹配就不再往下执行
  • 子类异常必须写在前面，父类异常写在后面
try {
    ...
} catch (NullPointerException e) {
    ...
} catch (Exception e) {
    ...
}
// 如果反了顺序，会编译错误

情况finally 是否执行

✅ Java 集合框架详解（List / Set / Map）

Java 的集合框架主要用来存储、操作对象数据，分为以下几大核心接口和实现类：


            ┌──────── Collection（接口） ────────┐
            ↓                                     ↓
        List（有序，可重复）                 Set（无序，不重复）
            ↓                                     ↓
     ArrayList, LinkedList              HashSet, TreeSet, LinkedHashSet

            ┌──────── Map（接口） ───────────────┐
            ↓                                     ↓
     HashMap, TreeMap, LinkedHashMap      （键值对结构）

一、Collection 接口体系（List / Set）

📌 Collection 是最顶层接口（Map 除外）

它有两个重要子接口：

接口名特点
List有序、允许重复元素
Set无序、不允许重复元素（自动去重）

二、List 接口及其实现类

📌 List：有序、可重复、可通过索引访问

实现类特点常用场景
ArrayList数组结构，查找快、增删慢查询多、增删少
LinkedList链表结构，插入/删除快频繁插入删除
Vector线程安全，但已过时基本不用了

✅ 常见用法：

List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("A");  // 允许重复

System.out.println(list.get(1));  // B

三、Set 接口及其实现类

📌 Set：无序、不允许重复

实现类特点
HashSet最常用，基于 Hash 表，元素无顺序
LinkedHashSet有插入顺序
TreeSet可排序（基于红黑树）

✅ 示例：

Set<String> set = new HashSet<>();
set.add("Java");
set.add("Java");  // 自动去重

for (String s : set) {
    System.out.println(s);
}

四、Map 接口及其实现类

📌 Map：键值对结构，key 不允许重复，value 可重复

实现类特点
HashMap最常用，无序，允许 null
LinkedHashMap有插入顺序
TreeMap按 key 排序
Hashtable线程安全，已淘汰
ConcurrentHashMap并发安全，高并发场景使用

使用示例：

Map<String, Integer> map = new HashMap<>();
map.put("Alice", 18);
map.put("Bob", 20);
map.put("Alice", 22);  // 会覆盖前面的值

System.out.println(map.get("Alice"));  // 22

✅ 泛型（Generics）与 迭代器机制（Iterator）

这是 Java 集合框架的高级基础，理解这部分你才能更好地使用 List<String>、Map<String, Integer>、增强 for 循环、Iterator 等功能。

一、泛型（Generics）

🔹 什么是泛型？

泛型就是给类、接口、方法传入“类型参数”，使代码在编译时就能检测类型安全，提升代码复用性。

🔹 为什么需要泛型？

✅ 没有泛型的写法（容易出错）：

List list = new ArrayList();
list.add("Hello");
list.add(123); // 编译不报错，但运行时类型不一致

String str = (String) list.get(0); // 强制类型转换

✅ 使用泛型后：

List<String> list = new ArrayList<>();
list.add("Hello");
// list.add(123); // ❌ 编译报错

String str = list.get(0); // 不用强转，安全

🔹 泛型语法形式

使用位置示例
类上class Box<T> { T data; }
方法上public <T> void print(T t)
接口上interface Comparable<T>

🔹 多种常见写法

List<String> list = new ArrayList<>();         // 常见
Map<String, Integer> map = new HashMap<>();    // 键值对类型
List<?> all = new ArrayList<Integer>();        // 通配符，未知类型

🔹 泛型的类型限制（通配符）

写法说明
?任意类型
? extends Number上限通配符（Number及其子类）
? super Integer下限通配符（Integer及其父类）

二、迭代器机制（Iterator）

🔹 什么是迭代器？

迭代器是一种对象，它提供了顺序访问集合中元素的方式，不需要暴露底层结构。

所有 Collection 类型都支持迭代器。

🔹 使用方式

List<String> list = new ArrayList<>();
list.add("A");
list.add("B");

Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String value = it.next();
    System.out.println(value);
}

🔹 方法说明

方法含义
hasNext()是否还有元素
next()获取下一个元素
remove()删除当前元素

🔹 不能使用 for 循环删除元素，必须用 Iterator

// ❌ 错误方式：直接使用 for 删除元素
for (String s : list) {
    if (s.equals("B")) {
        list.remove(s); // 会抛 ConcurrentModificationException
    }
}

✅ 正确方式：

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().equals("B")) {
        it.remove(); // 安全删除
    }
}

三、增强 for 循环（foreach）

✅ 本质是基于 Iterator 实现

for (String s : list) {
    System.out.println(s);
}

和：

Iterator<String> it = list.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}

是等价的，foreach 是语法糖。

四、泛型 + 迭代器综合应用示例

List<Integer> numbers = new ArrayList<>();
numbers.add(1);
numbers.add(2);
numbers.add(3);

Iterator<Integer> it = numbers.iterator();
while (it.hasNext()) {
    Integer n = it.next();
    if (n == 2) {
        it.remove();
    }
}
System.out.println(numbers); // 输出：[1, 3]

五、面试常见问题（中日英三语）

❓ Q1：泛型是编译期还是运行期有效？

  • 🇨🇳 编译期有效，运行时泛型信息被擦除（类型擦除 Type Erasure）
  • 🇯🇵 コンパイル時のみ有効、実行時は消される（型消去）

❓ Q2：为什么 foreach 删除元素会报错？

因为 增强 for 背后是使用 Iterator 实现的，但你手动修改集合（比如调用 list.remove()）就会破坏它的结构一致性，抛出 ConcurrentModificationException
