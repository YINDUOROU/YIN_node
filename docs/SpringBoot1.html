Spring & Spring Boot笔记1

第1章：Spring & Spring Boot 简介

1. 什么是 Spring？

Spring 是一个开源的 Java 企业级应用开发框架，旨在简化 Java EE 开发的复杂性。

Spring 的主要特点

  • 依赖注入（DI）：通过配置文件或注解注入依赖对象。
  • 面向切面编程（AOP）：支持业务逻辑与系统服务（如日志、安全等）分离。
  • 轻量级容器：自带 IoC 容器，可以管理 Bean 生命周期。
  • 事务管理：统一的声明式事务管理机制。
  • 模块化：包含多个模块，如 Spring Core、Spring MVC、Spring JDBC、Spring Security 等。

2. 什么是 Spring Boot？

Spring Boot 是基于 Spring 框架的快速开发框架，它简化了 Spring 项目的配置和部署。

核心目标

  • 简化配置（约定大于配置）
  • 内嵌服务器（如 Tomcat、Jetty）
  • 快速启动和部署（main() 方法一键运行）
  • 强大的自动配置（AutoConfiguration）

3. Spring 与 Spring Boot 的关系

项目SpringSpring Boot
框架类型全功能开发框架Spring 的快速开发工具包
配置方式手动配置较多（XML、注解）自动配置为主，支持微调
启动方式需要配置 web.xml、部署 war一键启动（内嵌 Tomcat）
学习曲线较陡（配置项繁琐）更友好（上手快）
使用场景精细控制复杂项目快速开发微服务、后台管理系统等轻量应用
✅ 简单理解：Spring 是底层框架，Spring Boot 是整合优化的快速开发工具。

4. Spring Boot 的主要特性（为什么它流行？）

  • 自动配置：自动推断项目所需的配置，极大地减少开发成本。
  • 快速开发：创建项目、运行服务只需几分钟。
  • 内嵌服务器：无需部署 WAR 包，直接运行。
  • 开箱即用：整合主流技术栈（JPA、Redis、Security、Swagger 等）。
  • 微服务友好：是 Spring Cloud 的基础，支持服务拆分与部署。

5. Spring Boot 项目适合谁用？

  • 学生 & 初学者：降低学习成本。
  • 后端开发者：快速构建 REST API。
  • 微服务架构团队：模块化拆分，快速部署。
  • 企业项目开发：高效、标准化、易维护。

6. Spring Boot 真实场景应用举例

  • 企业后台管理系统（CMS、ERP、CRM）
  • 电商订单处理系统
  • 医疗预约系统
  • 移动 App 的后端 API 服务
  • 微服务架构中的各个服务模块（如用户服务、订单服务等）

📚 小结

  • Spring 是基础框架，提供强大的依赖注入和模块化功能。
  • Spring Boot 是基于 Spring 的快速开发工具，适用于微服务和快速构建应用。
  • 掌握 Spring Boot 后，可以大大提升开发效率，简化配置和部署流程。

1. 什么是 REST 风格接口？

🔹 REST（Representational State Transfer）

一种通过 URL 路径 + HTTP 方法（GET/POST/PUT/DELETE）来表达操作资源的风格。

请求方式含义示例作用
GET查询获取数据
POST新增提交表单、创建数据
PUT修改更新已有资源
DELETE删除删除某个资源
REST 是一种风格，不是规范。Spring Boot 原生支持 REST 风格开发。

public class Result<T> {
    private Integer code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        Result<T> r = new Result<>();
        r.code = 200;
        r.message = "Success";
        r.data = data;
        return r;
    }

    public static Result<?> error(String msg) {
        Result<?> r = new Result<>();
        r.code = 500;
        r.message = msg;
        return r;
    }
    // getter/setter...
}

控制器中返回统一格式：

java

@GetMapping("/info")
public Result<String> getInfo() {

}

{

  "message": "Success",

1. 什么是依赖注入（DI：Dependency Injection）？

Spring 框架的核心思想就是 控制反转（IoC）+ 依赖注入（DI）。

💡 通俗理解：

以前我们写代码是自己 new 一个对象（主动），

现在由 Spring 帮我们创建并注入（被动）。

举例说明：

✅ 传统写法：

java

UserService userService = new UserService();

✅ Spring 写法（依赖注入）：

java

@Autowired
private UserService userService;

🔎 Spring 容器自动创建并注入，开发者不用手动 new。

2. 四大核心注解：声明组件

注解名用途描述作用层级
@Component万能组件注解所有通用类
@Service标注业务逻辑层组件Service 层
@Repository标注数据访问层组件DAO/Repository 层
@Controller标注 Web 控制层组件Controller 层

💡 以上四个注解本质上都是 @Component 的衍生，标注后 Spring 扫描时就会把它们注册为 Bean。

面试拓展：@Component 和 @Bean 区别？

项目@Component@Bean
用法用于类级别的自动扫描用于配置类中手动注册实例
书写位置类上方方法上方
场景自定义类、自动注入组件第三方类、手动构造复杂对象

3. @Autowired 注入依赖的三种方式

1）字段注入（最常见 ✅ 推荐学习期使用）

java
复制编辑
@Autowired
private UserService userService;

2）构造器注入（推荐实战中使用 ✅ 安全 + 可测试）

java
复制编辑
private final UserService userService;

@Autowired
public UserController(UserService userService) {
    this.userService = userService;
}
✅ 构造器注入的好处：
  • 支持单元测试 mock 注入
  • 不依赖反射，效率高
  • 更容易实现不可变性（final）

3）Setter 注入（较少使用）

java

private UserService userService;

@Autowired
public void setUserService(UserService userService) {
    this.userService = userService;
}

4. Spring 如何管理这些 Bean？

当项目启动时，Spring Boot 会：

  1. 扫描标注了注解的类（默认从主类所在包及子包）
  1. 实例化对象
  1. 放入 Spring 容器（ApplicationContext）
  1. 在需要用的地方通过 @Autowired 注入使用

5. 实战示例：注入一个服务组件

java

// service/UserService.java
@Service
public class UserService {
    public String greet() {
        return "你好，Spring Boot 学习者！";
    }
}

// controller/UserController.java
@RestController
@RequestMapping("/api")
public class UserController {

    @Autowired
    private UserService userService;

    @GetMapping("/greet")
    public String greetUser() {
        return userService.greet();
    }
}

访问：

bash

http://localhost:8080/api/greet

6. 拓展：Bean 的作用域

默认情况下，所有 Bean 是单例（singleton）的。

可选作用域：

  • singleton（默认）：容器中只创建一个实例
  • prototype：每次调用创建新实例
  • request（仅限 Web）：每次请求一个实例
  • session（仅限 Web）：每个会话一个实例

使用方式：

java

@Component
@Scope("prototype")
public class MyBean { }

❓ Q1：什么是依赖注入？Spring 是如何实现的？

✅ 回答要点：

依赖注入是将对象的依赖关系交给框架管理，由 Spring 容器统一创建和注入，避免了手动 new 对象。Spring 通过注解（如 @Autowired）结合 ApplicationContext 管理 Bean 实例。

❓ Q2：@Component 和 @Bean 有什么区别？用在什么场景？

✅ 回答要点：

@Component 是自动扫描注册组件，适用于我们自己写的类；而 @Bean 主要用于配置类中，显式地手动创建第三方类或复杂对象。两者作用相同，但使用方式不同。

❓ Q3：为什么推荐使用构造器注入而不是字段注入？

✅ 回答要点：

构造器注入更利于测试（支持 mock）、更安全（避免空指针）、不依赖反射，能构建不可变对象，符合最佳实践。

🧠 面试常问问题汇总

❓ Q1：JPA 和 MyBatis 的区别和使用场景？

✅ 回答要点：

JPA 是 ORM 框架，使用声明式接口和自动生成 SQL，适合快速开发；MyBatis 是半自动化框架，SQL 手写控制力强，适合复杂查询和大项目优化。

❓ Q2：Spring Boot 中如何使用 MyBatis？

✅ 回答要点：

引入 mybatis-starter，定义 Mapper 接口，配合注解或 XML 编写 SQL，并将接口注入使用即可。MyBatis 天然支持动态 SQL 和多表关联查询。

❓ Q3：Spring Boot 中如何配置多数据源？

✅ 回答要点：

定义多个 DataSource Bean + 配置 @Primary 主数据源 + 使用 @MapperScan(basePackages = "...") 指定 Mapper 路径，分别创建 SqlSessionFactory 即可。
